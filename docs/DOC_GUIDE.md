# 📘 사용 가이드라인

## 1. 시작하기

### 1.1 설치

```bash
pip install streamlit matplotlib numpy pandas CoolProp
```

### 1.2 실행

```bash
cd gap_simulator/
streamlit run gap_simulator_app.py
```

브라우저에서 `http://localhost:8501` 자동 오픈.

## 3. 기본 워크플로우

### 3.1 단일 케이스 해석

1. **사이드바** → 증발기/응축기/운전조건/냉매 설정
2. **▶ 해석 실행** 클릭
3. 탭에서 결과 확인
   - 🌡️ **A 냉방**: Cap Retention, Q, SHR, 손실 분해
   - 💧 **B 비말**: Flux, Risk Zone, Weber 수, Q_carry
   - 📊 **C 압강**: ΔP, f-factor, 분해 비율
   - 🖼️ **개략도**: Gap 물리 현상 시각화
   - 🔬 **Level 2**: 상관식 자동 선택 표시 + Tube-Segment 상세 해석
   - 📋 **데이터**: 수치 테이블 + CSV 다운로드

### 3.2 다중 케이스 비교

1. 첫 번째 조건 설정 → **+ 현재 설정 케이스 추가** 클릭
2. 설정 변경 → 다시 **+ 케이스 추가** (2회 이상)
3. **📈 비교** 탭에서 오버레이 Figure 확인
4. **🗑 전체 삭제**로 리셋

### 3.3 JSON 설정 파일

- **불러오기**: 📁 JSON 파일 → Browse files → 기존 JSON 선택 → 설정 자동 반영
- **저장하기**: 💾 현재 설정 JSON 저장 → 📥 다운로드
- CLI에서도 동일 JSON 사용 가능: `python run_single.py config.json`

## 4. 설정 가이드

### 4.1 HX 타입 선택

| 타입 | 용도 | 설정 항목 |
|------|------|----------|
| **FT** (Fin-Tube) | 가정용 에어컨/건조기 | 핀타입, FPI, 배열, Row×Col |
| **MCHX** (Micro-Channel) | 자동차/소형 기기 | 채널 크기, 루버 피치, N_ports |

### 4.2 핀 타입별 특성

| 핀 타입 | h_o 강화 | f 증가 | 적합 용도 |
|---------|---------|--------|----------|
| Plain | 기준 (1.0×) | 기준 | 일반 |
| Wavy | 1.2~1.5× | 1.3~1.5× | 제습 강화 |
| Louvered | 1.4~1.8× | 1.5~2.0× | 고성능 |
| Slit | 1.3~1.6× | 1.4~1.7× | 절충형 |

### 4.3 Gap Mode 선택

| 모드 | Seal Fraction | 물리 의미 |
|------|--------------|----------|
| **open** | 0 (고정) | 완전 개방 — 최대 재순환 |
| **semi** | 0.0~1.0 | 부분 밀봉 — 실제 건조기 대부분 |
| **sealed** | 1.0 (고정) | 완전 밀봉 — 이상적 기준 |

### 4.4 권장 설정 범위

| 파라미터 | 범위 | 기본값 | 비고 |
|----------|------|-------|------|
| T_amb | 20~35°C | 27°C | 실내 온도 |
| RH | 0.30~0.90 | 0.70 | 한국 여름: 0.65~0.85 |
| CMM | 3~20 | 6.75 | 가정용: 5~10 |
| FPI | 10~20 | 14 | 밀핀: 18~22 |
| T_evap | 2~15°C | 5°C | 에어컨: 5~7°C |
| Gap | 5~100mm | — | 건조기 실제: 10~30mm |

## 5. 결과 해석 가이드

### 5.1 Cap Retention (Module A)

- **90% 이상**: 양호 — Gap 영향 미미
- **80~90%**: 보통 — 밀봉 개선 권장
- **80% 미만**: 주의 — Gap 손실이 성능을 크게 저하

### 5.2 Risk Level (Module B)

| 등급 | Flux | 조치 |
|------|------|------|
| SAFE | < 1.0 | 비말동반 무시 가능 |
| CAUTION | 1.0~4.0 | 모니터링 권장 |
| DANGER | 4.0~12.0 | 설계 변경 필요 |
| SEVERE | ≥ 12.0 | 즉시 조치 (Gap 확대 or 풍속 저감) |

### 5.3 ΔP (Module C)

- **가정용 건조기**: 100~300 Pa 정상
- **가정용 에어컨**: 50~200 Pa 정상
- **Gap 기여**: 소간격(G<10mm)에서 30~40%, G>50mm에서 15% 이하

## 7. 🔬 Level 2 코일 모델

### 7.1 활성화

사이드바 → **🔬 Level 2 코일 모델** 펼치기 → 체크박스 활성화

### 7.2 설정 항목

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 냉매 유량 | m_ref [kg/s] — 압축기 또는 데이터시트에서 확인 | 0.00458 |
| 입구 건도 | x_in — 팽창밸브 출구 건도 (보통 0.15~0.25) | 0.22 |
| 유동 배열 | counter (역류) / parallel (병류) | counter |
| 증발 상관식 | auto (자동) / gungor_winterton / shah | auto |
| 세그먼트/튜브 | 튜브 1개당 분할 수 (5~20) | 10 |

### 7.3 결과 해석

- **상관식 표시**: 증발기/응축기 자동 선택된 상관식과 범위 경고
- **Row별 Q**: 어느 Row에서 열전달이 집중되는지 확인
- **Tube별 상세**: 건도(x), h_i, T_wall, 습면 비율
- **건도 차트**: 160개 세그먼트의 건도 분포 (파랑=이상, 주황=전이, 빨강=과열)

### 7.4 주의사항

- Level 2는 Module A/B/C 파이프라인과 독립적 (별도 탭에서 확인)
- 계산 시간: ~2초 (160 seg × 20 iter × 5 outer iter)
- m_ref, x_in은 시스템 수준 데이터 필요 (팽창밸브/압축기 출력)

## 8. 트러블슈팅

| 문제 | 해결 |
|------|------|
| CoolProp 설치 실패 | 없어도 동작 (근사 함수 fallback) |
| 한글 깨짐 | Windows: malgun.ttf 자동 탐색 |
| 포트 충돌 | `--server.port 8502` 추가 |
| 해석 느림 | gap_points를 15~20으로 줄이기 |
| MCHX 에러 | compute_mchx_geometry 미지원 조합 확인 |
